# 美国灵活用工平台对比分析报告（含 Jitjatjo × Dayforce 关系梳理）

日期：2026-01-30

> 说明：本报告基于公开互联网信息（官网/帮助中心/应用商店/新闻稿等）整理，可能随产品策略、合规与市场扩张而变化；不构成法律或合规意见。

## 一、结论速览（给决策者）

- **Jitjatjo 与 Dayforce 的关系**：Dayforce 在 2025-09-08 的官方新闻稿中披露，其“近期收购（recent acquisition）”了 **JJJ International**，该公司以 **Jitjatjo** 与 **Network** 两个品牌运营；其中 **Jitjatjo 提供 W-2 按需临时用工**，Network 提供前线用工的 **VMS（供应商管理系统）** 能力。此后，Jitjatjo 官网与 App 均显示其已成为 **Dayforce 家族/Dayforce Flex Work** 的一部分，体现出产品并入与品牌整合方向。
- **市场分层**：美国“灵活用工/按需排班”平台可粗分为三类：
  1) **W-2 数字化派遣/雇主代理（EOR/Staffing-as-a-Service）**（强调合规与管理，平台/其合作实体作为雇主）；
  2) **W-2/1099 混合的劳务撮合市场（Marketplace）**（覆盖更广但合规复杂度更高）；
  3) **以 1099 为主的任务/短时工平台**（更偏“微任务/外包”，企业侧管理与合规控制能力参差）。
- **选择建议（简版）**：
  - 若企业已经在用 Dayforce HCM，希望统一管理正式员工 + 临时工/外包工，并需要 VMS/供应商与费用控制：优先评估 **Dayforce Flex Work（含 Jitjatjo/Network 能力）**。
  - 若目标是“快速获得一支合规的 W-2 弹性小时工队伍”并希望外包招聘、背调、发薪、合规：重点看 **Indeed Flex、Bluecrew、Jitjatjo/Dayforce Flex Work**。
  - 若目标是“广覆盖 + 角色多样 + 可混合 1099/W-2”：可对比 **Instawork、Wonolo、WorkWhile**。

## 二、Jitjatjo × Dayforce：关系与整合路径

### 2.1 公开关系确认

- **Jitjatjo 自身表述**：Jitjatjo 官网（主页/关于/联系等）明确写明 “Jitjatjo is now part of the Dayforce family”。
- **Dayforce 官方披露**：Dayforce 在 2025-09-08 新闻稿中称，扩展 Dayforce Flex Work 的举措“紧随（follows）”其对 **JJJ International** 的“近期收购”，并说明 JJJ International 以 **Jitjatjo** 与 **Network** 两品牌运营：
  - Jitjatjo：**W-2 按需临时用工**（on-demand temporary staffing）
  - Network：前线用工的 **VMS**（vendor management system）
- **产品与品牌整合证据**：App Store 上的 **Dayforce Flex Work** 应用描述提到其是 “new app for Jitjatjo and Network users”，并支持用同一邮箱登录访问原团队与班次；同时标注此前为 “Ondemand by Jitjatjo”。

### 2.2 Dayforce Flex Work 的定位（与收购逻辑）

Dayforce 在 Flex Work 页面中强调其目标是：
- 将企业的多家派遣/用工供应商“**集中到一个系统**”中，并提供移动优先的平台，实现对计划、履约与绩效的实时可视化；
- 支持合规：包含合同条款、资质/证照跟踪与工人记录等能力，用于跨供应商优化成本与合规；
- “一次发布、多渠道触达”的发布与分发：可同时触达自有人才池、常用工人/团队、以及代理机构网络。

结合上述信息，可将收购逻辑理解为：
- **Jitjatjo** 补齐“自营/自有供给侧的 W-2 按需派遣”能力（尤其前线/小时工场景）；
- **Network** 补齐“供应商与项目/班次层面的 VMS 管理”能力；
- 最终以 **Dayforce Flex Work** 为统一入口，把“正式员工（HCM）+ 临时工（CWM/VMS + On-demand Staffing）”整合在同一平台中，强化企业端的可视化、费用控制与合规能力。

## 三、对比框架：美国灵活用工平台的关键差异维度

为便于横向比较，本报告用以下维度统一分析：

1) **用工模式（W-2 vs 1099 vs 混合）**：涉及税务、保险、工伤、加班、最低工资、雇佣关系等合规与成本结构。
2) **供给侧质量与履约机制**：筛选/背调/技能认证、评级、取消政策、补派机制、客服支持。
3) **企业端管理深度**：排班、考勤/定位、绩效评价、发薪/账单、费用与合同规则、报表分析、合规留痕。
4) **行业与岗位覆盖**：仓储物流、制造、零售、餐饮酒店、活动展会、保洁、文员等。
5) **系统集成能力**：是否提供 VMS、是否能与 HRIS/Payroll/Time 系统打通（或本身就是 HCM）。

## 五、平台逐一要点（更细的比较）

### 5.1 Dayforce Flex Work / Jitjatjo / Network

- **强项**：
  - 通过收购将 **W-2 按需派遣（Jitjatjo）+ VMS（Network）** 与 Dayforce 自身 HCM 叠加，形成“总劳动力（total workforce）”管理叙事。
  - Flex Work 页面强调合同条款、资质/证照跟踪与供应商绩效/开票等，适合更重的企业治理与费用控制。
- **可能的限制**：
  - 若企业不是 Dayforce 客户，集成与采购决策链更复杂；同时平台在不同州/行业的供给密度、履约 KPI 仍需按城市验证。

### 5.2 WorkWhile

- **强项**：
  - 企业侧卖点非常清晰：筛选/背调、可认证技能（叉车/食品处理/安保等）、按需/长期/转正、多行业覆盖。
  - 帮助中心显示其同时具备 1099 与 W-2 供给组织方式；并在加州推进非驾驶岗位的 W-2 化（公开原因与城市检察官办公室协议相关）。
- **风险/关注点**：
  - 用工分类在不同岗位/州差异较大，企业应明确每类班次的雇佣与保险责任归属（平台、合作 EOR、或企业自身）。

### 5.3 Instawork

- **强项**：
  - 角色覆盖广（官网列举仓储/餐饮/清洁/活动等），并允许在市场/角色条件允许时选择 1099 或 W-2 参与模型。
  - 帮助中心对 1099/W-2 的差异与税务路径解释较明确。
- **适配建议**：
  - 适合“跨城市、角色多、需要快速补人”的运营型企业；但应对用工分类做州与岗位级别的合规审查。

### 5.4 Indeed Flex

- **强项**：
  - 明确的 **W-2 员工供给**定位（对企业页面直接写明），并通过 PR 提到扩展市场与 VMS 产品（Flex Plus）。
  - 对求职者侧强调福利、奖金、以及同日/快速支付等增强留存机制。
- **适配建议**：
  - 适合偏重“合规 + 质量 + 可重复用工”的企业，尤其需要构建自有人才池与复用的场景。

### 5.5 Bluecrew（Employbridge 旗下）

- **强项**：
  - 明确主打 “W-2 Workforce-as-a-Service”，并在官网列出覆盖的全流程外包能力（招聘、背调、E-Verify、入职、排班、考勤、合规、薪资福利等）。
  - About 页面披露其覆盖州与市场数量，并强调背靠 Employbridge 的规模与能力。
- **适配建议**：
  - 适合仓储物流、制造等“波动用工 + 需要强合规与管理外包”的企业。

### 5.6 Wonolo

- **强项**：
  - “always-on staffing solution” 的表述更偏“平台 + 服务团队”的组合；面向企业页面写明可提供 W-2 与 1099 选项。
  - PR 显示其在推动 AI 驱动的岗位管理/匹配与透明化（用于提升效率与匹配质量）。
- **适配建议**：
  - 适合既有短期需求也有持续/转正需求的企业，特别是零售、制造、酒店、仓储等。

### 5.7 Shiftsmart

- **强项**：
  - 强调端到端的劳动力管理平台能力：来源、入职/培训、调度、绩效实时指标、沟通、快速支付等。
  - 帮助中心明确对某类伙伴的 1099 分类与税务提醒。
- **适配建议**：
  - 更适合“可被拆分为标准化班次/任务、需要规模调度”的项目型用工。

### 5.8 GigSmart（Get Workers）

- **强项**：
  - 强调低门槛、快速发布、在 App 内完成招人/管理/支付；App Store 描述强调“只为成功付费、无合同门槛”等。
- **适配建议**：
  - 更适合对系统整合要求不高、需要快速补人的中小企业或本地化需求。

## 六、企业选型建议（可落地的检查清单）

在实际选型时，建议把“灵活用工”拆成 6 个可验证问题（建议一城一验）：

1) **用工分类与责任边界**：每个岗位/州是 W-2 还是 1099？雇主是谁？工伤、失业保险、加班与最低工资责任由谁承担？
2) **履约与补派 SLA**：平均填补率、临时取消率、迟到/缺勤处理、替补速度、客服响应时间。
3) **供给质量与认证**：背调、技能证书、合规培训、现场安全机制、重复用工/人才池策略。
4) **费用结构透明度**：小时费率、加价/服务费、加班与节假日、取消费用、发票粒度、对账周期。
5) **系统能力**：是否支持 VMS（多供应商、合同规则、开票与 spend 控制）、是否有 API/与现有 HRIS/Time/Payroll 集成方案。
6) **合规与风控历史**：是否公开披露合规调整（例如某州从 1099 转向 W-2 的路径）、是否能提供审计材料与劳动合规支持。

## 七、W-2 与 1099 的法规与合规要点（美国）

> 重要澄清：**W-2/1099 本质是税务/信息申报表格**。口语中常用“W-2 工”“1099 工”来指代“雇员/独立承包商”，但法律判断并不取决于你发的是哪张表，而取决于你们的实际合作关系与适用的法律测试。本节仅做通用信息梳理，不构成法律意见。

### 7.1 概念澄清：W-2/1099 是表格，“雇员/承包商”是法律结论

- **W-2**：通常对应“雇员工资（wages）”信息申报（由雇主出具给雇员）。
- **1099-NEC**：通常对应“非雇员报酬（nonemployee compensation）”信息申报（企业向独立承包商等支付劳务报酬）。
- **常见误区**：
  - “我拿到 1099，所以一定是独立承包商”并不成立；
  - “合同写了 independent contractor，所以就不是雇员”也不成立。
- **没有一套统一标准**：税务（IRS）、工资工时（FLSA/DOL）、以及州法（失业保险、工伤、州劳动法）可能采用不同测试；同一合作关系在不同法规下的风险可能不同。

### 7.2 典型合规差异（企业视角，非穷尽）

| 维度 | W-2（雇员） | 1099（独立承包商） | 常见风险点 |
|---|---|---|---|
| 税务与申报 | 雇主通常承担工资税预扣/申报义务，并向雇员出具 W-2 | 企业通常不代扣个税与工资税，但可能需要出具 1099-NEC（常见门槛：全年支付非雇员报酬 >= $600） | “不想上 W-2”不是合规理由；以 1099 规避工资税风险很高 |
| 最低工资/加班（FLSA） | 通常受 FLSA/州法保护 | 通常不适用 FLSA 最低工资/加班保护 | 业务上按雇员方式管理（排班、考勤、纪律）但用 1099，极易被认定为误分类 |
| 保险与福利 | 可能涉及失业保险、工伤保险、福利计划资格等（依州法与雇主计划） | 通常自行承担保险与经营成本 | 发生工伤/争议时，责任边界不清会迅速放大成本与声誉风险 |
| 管理方式 | 用工方可更强控制（工作方法、流程、培训、绩效） | 更强调“结果导向”，承包商自主决定工作方式 | “控制权”是多数测试的关键因素；强控制 + 1099 组合风险最高 |

> 注：1099 申报存在大量细节（例如通过信用卡/第三方支付网络的付款通常不由付款方申报 1099-NEC），应以 IRS 说明为准。

### 7.3 IRS（联邦税务）视角：如何判断雇员 vs 独立承包商

- **普通法控制测试（Common law rules）**：IRS 的关键点是“你是否有权指挥/控制对方如何做事”，并综合以下三大类要素：
  - 行为控制：培训、流程、监督、对工作方式的要求；
  - 财务控制：费用承担、工具设备、付款方式、盈亏机会；
  - 双方关系：合同条款、福利、关系持续性、工作是否为业务关键部分。
- **不确定时的机制**：可提交 **Form SS-8** 请求 IRS 就联邦税务目的做身份判定（通常耗时较长，且应考虑对后续一致性与审计的影响）。
- **Section 530 relief（历史性救济）**：在满足严格条件时，企业可能获得针对就业税的有限救济；适用与否高度依赖事实与留档。

### 7.4 DOL / FLSA（联邦工资工时）视角：经济现实测试与 2024 规则的“执法现状”

- **经济现实（Economic Reality）**：FLSA 下的核心问题是：劳动者在经济上是否依赖用工方，还是在经营自己的独立业务。合同标签、是否领取 1099、是否“行业惯例”都不构成决定性依据。
- **2024 最终规则（六因素）**：DOL 的最终规则聚焦六类因素（如利润/亏损机会、投资、关系持续性、控制、工作是否为业务关键、技能与主动性），强调综合判断、无单一决定性因素。
- **2025 执法口径（WHD）**：根据 **Field Assistance Bulletin 2025-1**（2025-05-01），WHD 在自身调查/执法中不再适用 2024 规则，而是回到与长期判例一致的经济现实分析；同时提示该规则仍可能在私人诉讼中继续作为依据（最终由法院决定）。

### 7.5 州法示例：加州 AB5（ABC test）的要点（高度概括）

- **基本框架（ABC test）**：加州对很多情形推定劳动者为雇员，除非用工方能证明同时满足 A、B、C 三项要件（例如：不受控制、工作不在雇主常规业务范围内、劳动者独立经营同类业务等）。
- **豁免/例外很多**：部分职业、B2B 场景等可能豁免或改用其他测试（例如 Borello）；实际适用需逐项对照。
- **对灵活用工平台的现实含义**：在“平台撮合 + 企业强管理 + 前线岗位”的组合下，加州等严格州更容易推动 W-2 化（或通过派遣/EOR）以降低误分类风险。

### 7.6 对企业采购/使用灵活用工平台的合规建议（非法律意见）

- **把责任链条写清楚**：合同中明确“谁是雇主/记录雇主（EOR）”、谁负责预扣、W-2/1099 发放、工伤/失业保险、背景调查与合规培训。
- **避免“像雇员一样管理 1099”**：如果需要固定排班、现场管理、统一流程与纪律处分，通常更适合采用 W-2（或通过派遣/EOR）模式。
- **做州级别差异化策略**：至少对加州、马萨诸塞、新泽西等（常见更严格州）单独评估用工分类、合同模板与运营流程。
- **别忽视共同雇主风险**：即便劳动者是平台/派遣机构的 W-2 雇员，企业现场管理、排班与纪律控制仍可能在部分法规下触发共同雇主（joint employer）或连带责任讨论，需结合具体州法与合同条款评估。
- **保留证据与可审计记录**：岗位说明、合同、W-9/TIN、付款与发票、考勤与工作指令、合规培训、绩效与处罚记录等，建议形成可追溯链路。
